# CentOS_init

脚本使用方法：
将上述代码复制保存为一个文件，例如 setup_pentest_env.sh。
将文件上传到你的CentOS安全测试系统上。
给脚本执行权限：chmod +x setup_pentest_env.sh
以root用户或使用sudo执行脚本：sudo ./setup_pentest_env.sh
脚本会提示安装进度，并最终给出完成信息和后续步骤提示。
特别注意SET (Social-Engineer Toolkit) 的安装提示，克隆完成后，你需要手动进入 $TOOL_DIR/setoolkit 目录，然后运行 python3 setup.py 来完成最后的配置。
根据需要，将 $TOOL_DIR 下的工具路径或特定工具的可执行文件路径添加到系统的PATH环境变量中，方便直接在任何位置调用命令（脚本最后提供了示例）。
关于“灰产工具”和“免杀”的再次强调：
脚本中安装的工具（如sqlmap, hydra, nmap, SET等）本身是广泛用于安全测试的开源工具。它们并非专门的“灰产”工具，但可以被不法分子用于恶意目的。请务必严格遵守法律和公司规定，仅用于授权的测试。
“免杀”能力取决于很多因素，包括：
Payload类型： 是基于已知签名的可执行文件？是脚本（Python, PowerShell）？是Shellcode？

传输方式： 如何将Payload传递到目标？通过HTTP、DNS隧道、还是其他协议？
目标环境： 目标系统安装了哪些安全软件？其策略如何？是终端EDR还是网络IDS/IPS？
行为检测： Payload执行后的行为是否异常？是否触发了沙箱或行为分析？
持续对抗： 安全软件的检测规则不断更新，今天的免杀方法明天可能就失效。
